bool iu456_initialize(iu456_on_arrival_callback_t on_arrival_callback, iu456_on_removal_callback_t on_removal_callback, void *user_data, const char *path_to_bundle, iu456_error_details_t *error_details)¶

Initializes the library.

This function initializes the library. The user can also supply callbacks to be triggered whenever a compatible device is plugged in or removed. The user has also the ability to define where the ‘data’ folder will be located once installed.

Note
This function should be called before any other call to the library.
Note
The ‘on_arrival_callback’ will be also called for devices already plugged in.
Return
true if the operation performed correctly or false otherwise with the eventual error details object handle holding an error code and a message explaining the underlying error
Parameters
  • on_arrival_callback: [in]a callback called whenever a new device is plugged in; could be passed as NULL.
  • on_removal_callback: [in]a callback called whenever a device is unplugged; could be passed as NULL.
  • user_data: [in]a pointer that will be passed as second argument to both on_arrival_callback and on_removal_callback; could be passed as NULL.
  • path_to_bundle: [in]path to the zip or directory containing configuration data. If NULL, the builtin data will be used.
  • error_details: [out]error details object handle; could be passed as NULL.

bool iu456_initialize_with_device_property(iu456_on_arrival_callback_t on_arrival_callback, iu456_on_removal_callback_t on_removal_callback, void *user_data, const char *path_to_bundle, const uint8_t *rom_slave_address, uint32_t number_of_address, int production_address, int calibration_address, int device_index, iu456_error_details_t *error_details)¶

Initializes the library with device property.

This function initializes the library using device property. The user can also supply callbacks to be triggered whenever a compatible device is plugged in or removed. The user has also the ability to define where the ‘data’ folder will be located once installed. The user can also supply EEPROM salve address and where the production data and calibration data located on th EEPROM. The user can also select one from multiple devices.

Note
This function should be called before any other call to the library.
Note
The ‘on_arrival_callback’ will be also called for devices already plugged in.
Note
When using ‘device_index’, ‘on_arrival_callback’ after initialize is disabled, ‘on_removal_callback’ is valid only for the selected device.
Return
true if the operation performed correctly or false otherwise with the eventual error details object handle holding an error code and a message explaining the underlying error
Parameters
  • on_arrival_callback: [in]a callback called whenever a new device is plugged in; could be passed as NULL.
  • on_removal_callback: [in]a callback called whenever a device is unplugged; could be passed as NULL.
  • user_data: [in]a pointer that will be passed as second argument to both on_arrival_callback and on_removal_callback; could be passed as NULL.
  • path_to_bundle: [in]path to the zip or directory containing configuration data. If NULL, the builtin data will be used.
  • rom_slave_address: [in]a pointer that the rom slave adresses, If NULL, the default value will be used.
  • number_of_address: [in]element count of rom_slave_address, If 0, the default eeprom slave address will be used.
  • production_address: [in]a production data location address on the EEPROM, can be passed as -1 for default location address.
  • calibration_address: [in]a calibration data location address on the EEPROM, can be passed as -1 for default location address.
  • device_index: [in]the index of specified device to initialize; can be passed as -1 for connecting all devices.
  • error_details: [out]error details object handle; could be passed as NULL.

bool iu456_get_last_frame(iu456_handle_t *handle, const iu456_frame_t **frame_data, int32_t timeout, iu456_error_details_t *error_details)¶

Get data for the next frame.

This function will wait for the next frame and populate the frame_data pointer with the new data. The client can pass a timeout in milliseconds after which the function will return if no new data has been received.

Note
The value of frame_data is guaranteed until the next call to iu456_get_last_frame() or a call to iu456_stop().
Note
Even though an infinite timeout can be passed. The stream may be interrupted after a certain amount of time if the device is not streaming.
Return
true if the operation performed correctly or false otherwise with the eventual error details object handle holding an error code and a message explaining the underlying error
Parameters
  • handle: [in]pointer to an already allocated object handle.
  • frame_data: [out]pointer to an unallocated const iu456_frame_t *.
  • timeout: [in]a timeout in milliseconds; can be passed as -1 for infinite timeout.
  • error_details: [out]error details object handle; could be passed as NULL.

bool iu456_update_default_trigger_factor(iu456_handle_t *handle, int configuration_uid, int number_of_frames, iu456_trigger_sampling_t sampling_mode, iu456_error_details_t *error_details)¶

Specify the trigger upsampling or downsampling factor.

When choosing iu456_trigger_sampling_downsample, number_of_frames specifies the number of triggers to be ignored. For instance if number_of_frames is equal to 1, the trigger frequency will be halved.

When choosing iu456_trigger_sampling_upsample, number_of_frames specifies the number of frames to capture between 2 triggers. For instance if number_of_frames is equal to 2, the trigger frequency will be doubled.

The default values are 1 frame and iu456_trigger_sampling_upsample.

Return
true if the operation performed correctly or false otherwise with the eventual error details object handle holding an error code and a message explaining the underlying error
See
iu456_update_default_trigger_delay, iu456_get_trigger_factor_range
Parameters

  • configuration_uid: [in]the configuration uid for which to update the default.
  • number_of_frames: [in]the number of frames to capture.
  • error_details: [out]error details object handle; could be passed as NULL.

typedef struct iu456_frame_t iu456_frame_t¶

An object providing the data for a received frame.

This structure holds the post processed depth and confidence as well as other meta data linked to the frame itself.

The confidence map reflects the quality of the data at a given pixel. The greater this value is, the lesser noise the data will have.

The depth map is the cartesian depth expressed in millimeters and uses values greater than 32000 to flag invalid values.

The x and y pointers provides access to the first 2 cartesian coordinates. This means that when using (x, y, depth), one gets a list of 3d vertices in camera space. Note that x and y maps are optional (see iu456_set_x_and_y_property_state()). When disabled, these 2 pointers will be set to nullptr.
